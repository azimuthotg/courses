# Project Overview
**Project Name:** Micro-LMS (Learning Management System)
**Description:** ระบบนำเสนอสื่อการเรียนรู้ออนไลน์ขนาดเล็ก สำหรับห้องสมุด สำนักวิทยบริการ มหาวิทยาลัยนครพนม รองรับการเข้าสู่ระบบด้วย NPU AD, การเรียนผ่านวิดีโอ YouTube, การทำแบบทดสอบ (Pre-test / Post-test) และการออกใบประกาศนียบัตร (E-Certificate) เมื่อผ่านเกณฑ์

---

## 1. Tech Stack & Environment
*   **Backend:** Django (Python)
*   **Database:** MySQL
*   **Frontend:** HTML + Tailwind CSS (via CDN for simplicity)
*   **Authentication:** `django-auth-ldap` (Connect to NPU AD)
*   **Static Files:** `whitenoise` (Served by Waitress WSGI)
*   **Deployment Server:** Windows Server (IIS as Reverse Proxy + NSSM + Waitress)

---

## 2. Server & Proxy Configuration (Critical)
ระบบจะถูก Deploy ภายใต้ Subpath บน IIS Reverse Proxy โดยมี URL คือ `https://lib.npu.ac.th/courses/`
**Django `settings.py` requirements:**
*   `FORCE_SCRIPT_NAME = '/courses'`
*   `USE_X_FORWARDED_HOST = True`
*   `SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')`
*   `STATIC_URL = '/courses/static/'` (or dynamic based on force script name)
*   `whitenoise.middleware.WhiteNoiseMiddleware` must be added.

---

## 3. Database Schema (Django Models)

**1. User (Custom User / AbstractUser)**
*   รองรับข้อมูลจาก AD (Username, First Name, Last Name, Email)

**2. Course (หลักสูตร)**
*   `title` (CharField), `description` (TextField), `thumbnail` (ImageField)
*   `require_post_test` (BooleanField): กำหนดว่าคอร์สนี้บังคับสอบวัดผลหรือไม่ (Default: False)
*   `is_active` (BooleanField): สถานะเปิด/ปิดคอร์ส

**3. Lesson (บทเรียน)**
*   `course` (ForeignKey -> Course)
*   `title` (CharField)
*   `youtube_video_id` (CharField): เก็บแค่ ID ของ YouTube
*   `order` (IntegerField): ลำดับบทเรียน

**4. Quiz / Question / Answer (แบบทดสอบ)**
*   `Quiz`: เชื่อมกับ Course, ระบุ `quiz_type` (Pre-test / Post-test)
*   `Question`: เชื่อมกับ Quiz, มี `text` (โจทย์)
*   `Answer`: เชื่อมกับ Question, มี `text` (ตัวเลือก) และ `is_correct` (BooleanField)

**5. UserProgress (ประวัติและสถานะการเรียน)**
*   เชื่อม User และ Course
*   `status` (CharField/Enum): 'In Progress' หรือ 'Completed'

**6. UserQuizAttempt (ผลการสอบ)**
*   เชื่อม User และ Quiz
*   `score_percentage` (FloatField)
*   `is_passed` (BooleanField)

**7. Certificate (ใบประกาศนียบัตร)**
*   เชื่อม User และ Course
*   `issued_at` (DateTimeField)

## 4. Business Logic & Rules

**การเรียนและการผ่านหลักสูตร:**
1.  **เกณฑ์การสอบผ่าน:** กำหนด Passing Score = 70% สำหรับ Post-test
2.  **การเปลี่ยนสถานะ (Status Tracking):**
    *   เมื่อผู้ใช้เริ่มเข้าดูบทเรียนแรก: เปลี่ยน `UserProgress.status` เป็น **"In Progress"**
    *   **เงื่อนไขการ Completed:** 
        *   *กรณี `require_post_test == False`*: ผู้ใช้ดูบทเรียนครบทุกอัน = **"Completed"**
        *   *กรณี `require_post_test == True`*: ผู้ใช้ต้องทำ Post-test ให้ได้คะแนน >= 70% เท่านั้น จึงจะเปลี่ยนเป็น = **"Completed"**
3.  **Certificate Generation:** ใบประกาศนียบัตรจะถูกสร้างขึ้นอัตโนมัติ (และอนุญาตให้ดาวน์โหลดเป็น PDF) เฉพาะเมื่อสถานะการเรียนของคอร์สนั้นเป็น **"Completed"** เท่านั้น

## 5. URL Routing & Views
ใช้ **Class-Based Views (CBV)** ในการพัฒนา
*   `/` -> `CourseListView` (Dashboard แสดงคอร์สที่เปิดสอน)
*   `/login/` -> AD Login View
*   `/course/<pk>/` -> `CourseDetailView` (รายละเอียด และปุ่มเริ่มเรียน)
*   `/course/<course_pk>/lesson/<lesson_pk>/` -> `LessonView` (แสดงวิดีโอ YouTube)
*   `/course/<course_pk>/quiz/<type>/` -> `QuizView` (ทำแบบทดสอบ Pre/Post)
*   `/certificate/<course_pk>/` -> `CertificateDownloadView` (Generate PDF e.g., using xhtml2pdf or WeasyPrint)
